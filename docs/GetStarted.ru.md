# Плагин Tuya — Начало работы

## Дополнительные материалы

После первого запуска используйте:

- `index.ru.md` — обзор модуля и терминов
- `Setup.ru.md` — подробная настройка режимов подключения и ключей
- `DevicesAndDPS.ru.md` — работа с устройствами, категориями и DPS
- `Troubleshooting.ru.md` — типовые ошибки и способы диагностики

## Предварительные требования

- Аккаунт в приложении [Tuya Smart](https://play.google.com/store/apps/details?id=com.tuya.smart) или [Smart Life](https://play.google.com/store/apps/details?id=com.tuya.smartlife)
- Устройства уже добавлены и работают в приложении
- Доступ в интернет для регистрации в облаке

## Шаг 1 — Создайте облачный проект Tuya IoT

1. Перейдите на **https://iot.tuya.com/** и зарегистрируйтесь (или войдите)
2. В меню слева: **Cloud** → **Development** → **Create Cloud Project**
3. Заполните:
   - **Project Name** — любое имя (например `osysHome`)
   - **Industry** — Smart Home
   - **Development Method** — Smart Home
   - **Data Center** — выберите ваш регион:
     | Код | Регион |
     |-----|--------|
     | `eu` | Западная Европа (Ирландия) |
     | `us` | Западная Америка |
     | `cn` | Китай |
     | `in` | Индия |
4. Нажмите **Create**

## Шаг 2 — Скопируйте Access ID и Access Secret

После создания проекта вы увидите:

- **Access ID / Client ID** — это ваш `Access ID`
- **Access Secret / Client Secret** — это ваш `Access Secret`

> Сохраните эти значения — они понадобятся в настройках плагина.

## Шаг 3 — Подключите API-сервисы

На странице проекта перейдите во вкладку **Authorize API Services** и убедитесь, что подключены:

- **IoT Core** (обязательно)
- **Smart Home Scene Linkage** (опционально, для сценариев автоматизации)
- **IoT Data Analytics** (опционально, для истории и статистики)

Если их нет в списке, нажмите **Add Authorization** и найдите нужные сервисы.

## Шаг 4 — Привяжите устройства к облачному проекту

1. На странице проекта перейдите в **Devices** → **Link Tuya App Account**
2. Нажмите **Add App Account**
3. Откройте приложение **Tuya Smart** или **Smart Life** на телефоне
4. Перейдите в **Я** → нажмите иконку сканирования (вверху справа) → отсканируйте QR-код с сайта
5. Подтвердите привязку — все устройства из вашего аккаунта станут доступны через Cloud API

## Шаг 5 — Настройте плагин

1. Откройте страницу администрирования плагина Tuya в osysHome
2. Нажмите кнопку **Настройки** (иконка шестерёнки)
3. Введите:
   - **Access ID** — из Шага 2
   - **Access Secret** — из Шага 2
   - **Region** — тот же код Data Center, что вы выбрали в Шаге 1 (`eu`, `us`, `cn`, `in`)
   - **Connection Mode** — рекомендуется: `Both` (облако + локальное управление)
   - **Poll Interval** — частота опроса статуса устройств (по умолчанию: 30 секунд)
4. Нажмите **Save**

## Шаг 6 — Обнаружение устройств

Нажмите **Discover Devices** — плагин загрузит все устройства из вашего аккаунта Tuya, включая их имена, категории и локальные ключи.

## Шаг 7 — Привязка DPS к системным объектам

1. Нажмите кнопку **Link** (иконка цепи) напротив устройства
2. Загрузится список DPS:
   - **Cloud** (зелёный бейдж) — реальные DPS из спецификации устройства (только те, что устройство действительно поддерживает)
   - **Reference** (жёлтый бейдж) — стандартные DPS из встроенного справочника
3. Для каждого DPS выберите:
   - **Object** — системный объект для привязки
   - **Property** — свойство для чтения/записи значений
   - **Method** — метод для вызова (опционально)
   - **Mode** — отметьте "R/O", чтобы сделать записываемый DPS доступным только для чтения
4. Нажмите **Save**

## Опционально — Локальное управление

Для более быстрого отклика без задержек облака плагин поддерживает прямое управление по LAN через `tinytuya`.

Требования для локального управления:
- Устройство должно быть в той же сети, что и сервер
- Необходим **Local Key** устройства (заполняется автоматически при обнаружении через облако)
- **IP-адрес** устройства (определяется автоматически через сканирование LAN или вводится вручную)

Используйте кнопку **Scan LAN** для поиска устройств в локальной сети.

## Опционально — Ручное добавление устройств

Если вы не хотите использовать облако, можно добавить устройства вручную:

1. Нажмите **Add Device**
2. Введите:
   - **Device ID** — из приложения Tuya (Информация об устройстве → Virtual ID) или с самого устройства
   - **Device Name** — понятное имя
   - **Local Key** — необходим для локального управления (можно получить через tinytuya wizard или Cloud API)
   - **IP Address** — IP-адрес устройства в локальной сети
   - **Category** — тип устройства (определяет доступные DPS-коды)
3. Нажмите **Add Device**

## Решение проблем

| Проблема | Решение |
|----------|---------|
| Облако показывает "Disconnected" | Проверьте Access ID, Secret и Region; убедитесь, что проект активен на iot.tuya.com |
| Обнаружено 0 устройств | Убедитесь, что вы привязали аккаунт приложения к облачному проекту (Шаг 4) |
| Устройство показывает "Offline" | Проверьте, включено ли устройство и подключено ли оно к Wi-Fi |
| Локальное управление не работает | Проверьте Local Key и IP; убедитесь, что устройство в той же сети |
| Список DPS пуст | Категория может быть "unknown" — отредактируйте устройство и укажите правильную категорию |
| Команды не отправляются | Проверьте, не отмечен ли DPS как Read-only; проверьте логи на ошибки |

## Лимиты Cloud API

Tuya IoT Platform имеет ограничения в зависимости от тарифного плана:

| | Trial (бесплатный) | Flagship | Corporate |
|---|---|---|---|
| **Цена** | Бесплатно | Платная подписка | Платная подписка |
| **Макс. устройств** | **50** | 75 000 | 200 000 |
| **Управляемых устройств** | **10** | 30 000 | 75 000 |
| **API вызовов/месяц** | ~**26 000** | ~224 млн | ~426 млн |
| **Сообщений/месяц** | ~**68 000** | ~568 млн | ~1 млрд |
| **Data Center** | **1** регион | 7 регионов | 7 регионов |
| **Хранение логов** | Нет | 72 часа | 168 часов |

### Ограничения Trial Edition

- **50 устройств** максимум в проекте
- **10 управляемых** — только 10 устройств могут получать команды через Cloud API
- **~26 000 API вызовов/месяц** — при опросе 1 устройства каждые 30 секунд это ~86 400 вызовов/месяц, т.е. хватит примерно на **9 устройств** при постоянном опросе
- **При исчерпании лимита сервис приостанавливается** до следующего месяца (без дополнительных начислений)
- **Коммерческое использование запрещено**

### Рекомендуемая конфигурация

Используйте **Connection Mode: Both** (облако + локальное) для минимизации расхода API:

| Функция | Канал | Расходует API лимит? |
|---------|-------|----------------------|
| Обнаружение устройств | Облако | Да (разово) |
| Спецификация DPS | Облако | Да (разово для каждого устройства) |
| Push-уведомления (Pulsar) | Облако | Сообщения, не API вызовы |
| Опрос статуса | **Локальная сеть (LAN)** | **Нет** |
| Отправка команд | **Локальная сеть (LAN)** | **Нет** |
| Резервные команды | Облако | Да |

При использовании локального управления как основного канала, облачное API расходуется только на обнаружение и push-уведомления, что позволяет оставаться в рамках бесплатного плана даже с десятками устройств.
