# Плагин Tuya — Документация

Этот раздел помогает быстро понять, как устроен модуль `Tuya`, какой режим подключения выбрать, как добавить устройства и как связать DPS со свойствами системы.

## Что умеет модуль

- Подключение к Tuya Cloud через `Access ID` и `Access Secret`
- Локальное управление устройствами по LAN через `tinytuya`
- Автообнаружение устройств из аккаунта `Tuya Smart` / `Smart Life`
- Ручное добавление устройств по `Device ID`, `Local Key` и IP
- Получение `Local Key` из учетной записи приложения прямо из интерфейса модуля
- Привязка DPS к объектам и свойствам системы
- Резервирование каналов: локальное управление как основное, облако как запасной путь

## Как читать эту документацию

- `GetStarted.ru.md` — самый быстрый путь к первому запуску
- `Setup.ru.md` — подробная настройка облака, локального управления и режимов подключения
- `DevicesAndDPS.ru.md` — добавление устройств, сканирование LAN, категории и привязка DPS
- `Troubleshooting.ru.md` — типовые ошибки, проверка настроек и практические советы

## Основные термины

- `Tuya Cloud` — облачный API Tuya, нужен для обнаружения устройств, получения их описания и резервного управления
- `LAN / Local` — прямое управление устройством по локальной сети без обращения в облако
- `Device ID` — уникальный идентификатор устройства Tuya
- `Local Key` — ключ для локального управления устройством
- `Category` — тип устройства в Tuya, определяющий набор типовых DPS-кодов
- `DPS` — Data Points, отдельные параметры устройства: питание, яркость, температура, режим и т.д.
- `Linked App User UID` — UID пользователя приложения `Tuya Smart` / `Smart Life`; помогает, если облако подключено, но обнаружение возвращает 0 устройств

## Рекомендуемый сценарий для большинства пользователей

1. Настройте облачный проект Tuya и подключите к нему аккаунт приложения.
2. Включите режим `Both (Cloud + Local)`.
3. Выполните `Discover Devices`.
4. Нажмите `Scan LAN`, чтобы модуль нашел IP-адреса устройств в локальной сети.
5. Если у устройств еще нет `Local Key`, используйте `Get local keys`.
6. Откройте `Link DPS` для каждого важного устройства и свяжите нужные точки со свойствами системы.

## Какой режим подключения выбрать

- `Cloud only` — если нужен только облачный доступ и локальная сеть недоступна
- `Local only` — если все устройства уже известны, у них есть `Local Key`, IP-адреса и не нужен облачный поиск
- `Both (Cloud + Local)` — рекомендуемый режим:
  - облако используется для обнаружения, описания DPS и резервного доступа
  - локальная сеть используется для быстрых команд и опроса статуса

## Что важно понимать заранее

- Не все устройства одинаково хорошо работают локально: многое зависит от прошивки.
- Ошибки `904`, `905`, `914` не всегда означают неверный ключ или протокол.
- Для локального управления устройство и сервер должны находиться в одной сети или в маршрутизируемых подсетях без блокировки TCP/UDP.
- Категория устройства влияет на удобство привязки DPS, но не заменяет реальные данные устройства из облака.

## Безопасность

- `Access Secret` и `Local Key` дают доступ к управлению устройствами.
- Не публикуйте их в скриншотах, логах и общих репозиториях.
- При использовании функции `Get local keys` учетные данные приложения применяются только для запроса и не должны использоваться как постоянная замена облачной настройке.
